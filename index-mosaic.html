运营管理

推荐工作流（轻量、好管）

在项目里建内容收集文件夹（默认 content/），图片按编号命名 0001.jpg……再配 CSV。格子只显示图，点开看文案和音视频。

CSV 表头（五列，UTF-8）：

编号,文案,图片url,音频url,视频url

  • 图片url 示例：./content/0001.jpg（与文件夹名、文件名一致）。也可用 https 图链。
  • 音频url / 视频url 不需要就整列留空。
  • 用 Excel 导出时请选「CSV UTF-8（逗号分隔）」；若导入显示成功 0 条，多半是编码不是 UTF-8，表头「编号」读不到。
  1. 图放进 content/，表按上行表头填写，另存 CSV。
  2. 分区③导入 CSV → 分区⑤保存或下载 data.json。
  3. 上线：网页 + data.json + content/ 整夹一起部署。

① 主视觉底图（整页一张）

② 格子内容对应关系（心里有个表）

内容 必填？ 写在哪
文案 每格要 CSV「文案」列
配图 每格要 CSV「图片url」，如 ./content/0001.jpg
音频 / 视频 随意 CSV「音频url」「视频url」，空 = 没有

③ 上传格子 CSV（推荐先做这一步）

表头：编号,文案,图片url,音频url,视频url。参考项目里 sample-import.csv。

④ 批量选本地图（可选，与 CSV 路径一致）

若 CSV 里写的是 ./content/0001.jpg，请把「内容文件夹」设为 content（或你的文件夹名），再把图片按编号命名放进对应目录后，在下方多选。也可用命令行 node scripts/generate-data-from-media.mjs content 把磁盘上的图合并进 data.json。

文件名须为四位编号：0001.jpg。部署时网站目录里要有同名文件夹与文件。

⑤ 保存 / 换电脑 / 备份

导入 data.json 会覆盖当前格子数据。改完务必保存。
