# 换图与发布：傻瓜步骤（以后照着做）

> 目标：**换完图 → 主视图马赛克快、点开弹窗仍高清**；**别漏步骤**。

---

## 一、先搞清楚：网站用哪几张「图」

| 东西 | 放哪 / 写什么 | 作用 |
|------|----------------|------|
| **主视觉大图** | 后台「上传底图」或 `data.json` 里的 `heroImage` | 整块背景 |
| **每一格的配图（高清）** | 文件夹 `content/`，文件名 **`0001.jpg`**～**`1000.jpg`**（或 png/webp 等，见下） | **弹窗里的大图**、保存原图 |
| **每一格的小图（可选但强烈推荐）** | 文件夹 **`content/thumbs512/`**（与 `app.js` 里 `CONTENT_THUMB_SUBDIR` 一致），文件名 **`0001.webp`**～**`1000.webp`** | **主视图马赛克小格**，体积小、加载快 |
| **格子文案等** | `data.json` 或 CSV 再导入 | 点开格子看到的文字 |

**CSV 只有一列图片时**：那一列写 **`./content/0001.jpg`** 这种路径即可，**不用**多写一列小图；小图靠下面「第 3 步脚本」自动生成到 **`content/thumbs512/`**（默认；旧版曾为 `thumbs/`）。

---

## 二、换「格子配图」时你要做的事（按顺序打勾）

### 第 1 步：备份（强烈建议）

- [ ] 复制整个 **`content/`** 到别处（或压缩一份 zip），避免改错回不来。

### 第 2 步：替换原图（高清）

- [ ] 把新图放进 **`content/`**。
- [ ] **文件名必须是四位编号**，和格子一致，例如：`0426.jpg`、`0001.png`（与 `data.json` / CSV 里该格编号一致）。
- [ ] 若你删掉了某编号文件，记得 **`data.json` 里对应条目**也要改或删，否则点开还会去找旧路径。

### 第 3 步：重新生成「小图」给马赛克用（换原图后必做）

不跑这一步：**主视图可能还是旧小图**，或只能退回加载大原图（慢）。

1. 打开 **PowerShell**（或终端）。
2. 进入项目里的 **`scripts`** 文件夹（路径按你电脑实际改，有中文要加引号）：

   ```powershell
   cd "你的项目路径\scripts"
   ```

3. **第一次**在这台电脑上才需要装依赖（以后换图不用重复装）：

   ```powershell
   npm install
   ```

4. **生成 / 更新** `content/thumbs512/` 里的小 webp（在 `scripts` 目录里执行）：

   ```powershell
   node generate-content-thumbs.mjs ..\content
   ```

   - 想小图再糊一点、文件更小：把长边改小，例如 **384**：

     ```powershell
     node generate-content-thumbs.mjs ..\content 384
     ```

   - **以前已经生成过小图**，但这次想**强制按新原图全部覆盖**（同目录覆盖；若 Windows 报 EPERM，可改用新目录 **`--out=thumbs512b`** 并在 `app.js` 里把 `CONTENT_THUMB_SUBDIR` 改成同名）：

     ```powershell
     node generate-content-thumbs.mjs ..\content 512 --force
     ```

   - 写到别的子目录（与 `app.js` 中 `CONTENT_THUMB_SUBDIR` 一致）：

     ```powershell
     node generate-content-thumbs.mjs ..\content 512 --out=thumbs512
     ```

5. 执行完看窗口提示：应有「完成」「失败」数量；**失败**多为某张原图损坏或格式怪异。

**本步打勾：**

- [ ] 已执行 `node generate-content-thumbs.mjs ...`（必要时带 `--force`）。
- [ ] 打开文件夹 **`content/thumbs512/`**（或你 `--out=` 指定的目录），确认对应编号 **`.webp`** 存在且修改时间是最新的。

### 第 4 步：`data.json` / CSV 是否要对上

- [ ] 若你只**替换文件、没改编号和路径**：一般 **不用**改 `data.json`。
- [ ] 若你**改了扩展名**（例如从 `0001.jpg` 改成 `0001.webp`）：要在 **`data.json`**（或重新导 CSV）里把该格的 **`imageUrl` / 图片 url** 改成新路径。
- [ ] 若你用后台 **「导入 CSV」**：导入后记得 **「保存到当前浏览器」或「下载 data.json」**，部署要用最新那份。

### 第 5 步：本地打开网页自查（别省）

- [ ] 用浏览器打开 **`index.html`** 或本地起的静态服务，看：
  - [ ] **缩小看整幅**：马赛克小格是否更新、加载是否明显变快。
  - [ ] **点几格**：弹窗里大图是否清晰、是否仍是你要的那张。
- [ ] 若小格灰很久：按 **F12 → Network** 看 `content/thumbs512/xxxx.webp`（或你配置的子目录）是否 **404**（多半是第 3 步没跑或 `CONTENT_THUMB_SUBDIR` 与生成目录不一致）。

### 第 6 步：上传到 GitHub（线上/GitHub Pages）

- [ ] 在**项目根目录**用 Git 提交（至少要包含你改过的 **`content/`**（含 **`content/thumbs512/`** 等小图目录）、**`data.json`** 若有改）：

  ```powershell
  cd "你的项目路径"
  git add content data.json
  git status
  git commit -m "chore: 更新配图与 thumbs"
  git push origin main
  ```

- [ ] **`data.json` 若在仓库根目录**，也要 `git add data.json`（别只推图忘了稿）。
- [ ] 推送后等 **GitHub Actions / Pages** 构建完成（通常 1～3 分钟），再打开线上链接 **强制刷新**（Ctrl+F5）或无痕窗口测一次。

**容易忘：**

- [ ] 只推了 **`content/`** 原图，**忘了推小图目录（如 `content/thumbs512/`）** → 线上小格还是旧的或很慢。  
- [ ] 只改了本地 **`data.json`**，**没提交 / 没推送** → 线上格子数据还是旧的。

---

## 三、只换「主视觉大图」时（不动千格）

- [ ] 后台上传底图，或改 **`data.json` 顶层 `heroImage`** 为新的图片路径。
- [ ] 保存 / 导出 `data.json` → 提交推送（同上）。
- [ ] **不必**重跑 `generate-content-thumbs`（除非你也换了格子图）。

---

## 四、可选：想让访客「重新走一遍开幕仪式」

改 **`app.js`** 里的 **`SITE_DEPLOY_REVISION`**（日期或版本号字符串），保存后一并提交推送。  
（会清掉当前浏览器里与「仪式」相关的 session 标记，具体逻辑以代码注释为准。）

- [ ] 需要时再改，不是每次换图都要做。

---

## 五、遇到问题先对照这张表

| 现象 | 优先检查 |
|------|-----------|
| 弹窗图还是旧的 | `data.json` 里该格 `imageUrl` / 是否已部署最新 `data.json` |
| 马赛克小格旧 / 慢 | 是否跑过 **`generate-content-thumbs`**；**`content/thumbs512/`**（或当前子目录）是否上传；编号是否四位 |
| 小格一直灰 | 浏览器 Network 里 `thumbs` 或原图是否 **404**；文件名与编号是否一致 |
| 本地快、线上慢 | 是否文件太大；是否没推 `thumbs`；网络与 GitHub Pages 缓存（强刷） |

## 六、最短「换图 checklist」（复制到备忘录里用）

```
[ ] 备份 content
[ ] 替换 content/ 下对应编号原图
[ ] cd scripts → npm install（仅新机）
[ ] node generate-content-thumbs.mjs ..\content（换图后必跑；同目录覆盖加 --force）
[ ] 检查 content/thumbs512/ 里 webp 是否更新（与 app.js CONTENT_THUMB_SUBDIR 一致）
[ ] 改 data.json / CSV 的话已保存且路径对
[ ] 本地打开页面验：整图 + 点开弹窗
[ ] git add content data.json（按需；小图在 content/thumbs512/ 下会一并进 content）
[ ] git commit && git push
[ ] 等 Pages 构建完，线上强刷验证
```
